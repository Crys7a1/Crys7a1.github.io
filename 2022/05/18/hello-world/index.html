请求转发与重定向的异同

  1. 都可以实现访问一个资源时转向当前应用资源
  2. 请求转发-一次请求一次响应，重定向-两次请求两次响应
  3. 一般情况使用请求转发，减少浏览器对服务器的访问，减轻服务器的压力
  4. 改变浏览器地址栏，更改刷新功能使用重定向

Cookie 与 Session 的区别

1.Cookie 客户端技术-把信息存储在客户端
Session 服务端技术-把数据存储在服务端 2. Cookie 是基于 Http 协议中的 Set-Cookie 响应头和 Cookie 请求头进行工作的。
HttpSession 是基于特殊的 Cookie 工作的 3.浏览器对 Cookie 有严格的限制，对 Session 没有限制
